Analysis Domain

Reverse Engineering

바이너리의 구조를 파헤치고 악성코드의 은닉된 로직을 역추적하는 분석 기술

Written by 권도영

1. 리버스 엔지니어링의 본질

리버싱(Reversing)은 완성된 바이너리(실행 파일)를 디스어셈블(Disassemble) 및 디컴파일(Decompile)하여 소스 코드가 없는 상태에서도 프로그램의 내부 로직과 알고리즘을 파악하는 역공학 기술입니다. 주로 악성코드(Malware) 분석, 소프트웨어 취약점 탐지(0-day), 침해사고 원인 분석에 핵심적으로 사용됩니다.

2. Windows PE (Portable Executable) 구조 분석

Windows 환경에서 실행되는 파일(.exe, .dll, .sys)의 내부 포맷입니다. 리버서는 이 PE 헤더를 분석하여 프로그램이 메모리에 어떻게 적재되는지 파악합니다.

1. DOS Header & Stub

호환성을 위한 영역입니다. 파일의 맨 앞 2바이트는 항상 MZ (4D 5A)로 시작하며, DOS 환경에서 실행 시 "This program cannot be run in DOS mode"라는 메시지를 출력합니다.

2. NT Headers

PE 파일의 핵심 정보가 담겨있습니다. PE (50 45 00 00) 시그니처로 시작하며, File Header와 Optional Header로 구성됩니다. (진입점 주소인 OEP(Original Entry Point) 정보 포함)

3. Section Headers

각 섹션의 속성(크기, 위치, 권한)을 정의합니다. 대표적으로 .text(코드), .data(초기화된 데이터), .rdata(읽기 전용 데이터) 등이 있습니다.

4. IAT / EAT (Import/Export Table)

프로그램이 어떤 라이브러리(DLL)의 어떤 API 함수를 사용하는지(IAT), 또는 외부로 제공하는지(EAT) 기록된 테이블입니다. 악성코드 행위 분석의 첫 단추입니다.

3. 어셈블리어와 Calling Convention

C/C++ 함수가 호출될 때 인자(Argument)를 전달하고 스택(Stack)을 정리하는 규칙입니다. 디스어셈블된 코드를 볼 때 이 규약을 알아야 함수 분석이 가능합니다.

cdecl (C Declaration)

  • 주로 C 언어에서 사용되는 기본 규약입니다.
  • 인자를 스택에 푸시할 때 오른쪽에서 왼쪽 순서로 넣습니다.
  • 호출자(Caller)가 스택을 정리합니다. (예: add esp, 8)
  • 가변 길이 인자(예: printf) 처리에 적합합니다.

stdcall (Standard Call)

  • Windows API(Win32 API)에서 기본적으로 사용하는 규약입니다.
  • 인자는 동일하게 오른쪽에서 왼쪽으로 푸시합니다.
  • 피호출자(Callee)가 스택을 직접 정리하고 반환합니다. (예: ret 8)
  • 코드 크기가 상대적으로 작아집니다.

4. 고급 분석 방해 기법 (Anti-Reversing)

악성코드 분석가들을 속이고 지연시키기 위해 공격자들이 사용하는 기술들입니다.

Anti-Debugging API

IsDebuggerPresent(), CheckRemoteDebuggerPresent() 같은 API를 호출하여 디버거 환경을 탐지하고, 탐지 시 악성 행위를 숨기거나 강제 종료합니다. 리버서는 이 API의 리턴값을 조작하여 우회합니다.

PEB 조작 및 우회

Process Environment Block(PEB)의 BeingDebugged 플래그나 NtGlobalFlag를 직접 참조하여 디버깅을 탐지합니다. 동적 분석 시 해당 메모리 값을 강제로 0으로 변조(Patch)해야 분석이 가능합니다.

Anti-VM / Sandbox Evasion

VMware, VirtualBox 등의 가상머신 또는 샌드박스 환경에서 실행 중인지 CPUID 명령어나 특정 레지스트리 키, MAC 주소를 통해 확인하고 분석 환경을 회피합니다.

Packing & Obfuscation

UPX, Themida 같은 패커(Packer)를 사용해 원본 코드를 암호화하여 정적 분석을 막습니다. OEP(Original Entry Point)를 찾아 메모리를 덤프하는 언패킹(Unpacking) 기술이 필요합니다.

5. 핵심 리버싱 도구 (Tools)

IDA Pro / Ghidra

전 세계 표준 정적 분석 도구입니다. 바이너리를 어셈블리 코드로 변환하고 제어 흐름 그래프(CFG)를 제공하며, Ghidra는 강력한 디컴파일러(C 코드 복원) 기능을 무료로 제공합니다.

x64dbg

Windows 환경에서 가장 널리 쓰이는 오픈소스 동적 디버거입니다. 레지스터 변화를 실시간으로 확인하고 중단점(Breakpoint)을 설정하여 메모리를 분석합니다.
