Defensive Domain

System Hardening

인프라의 공격 표면(Attack Surface)을 최소화하고 심층 방어(Defense in Depth)를 구현하는 인프라 보안 핵심 기술

Written by 조하율

1. 시스템 하드닝과 심층 방어 (Defense in Depth)

시스템 하드닝(System Hardening)은 OS, 웹 서버, 데이터베이스, 클라우드 인프라 전반에서 기본(Default) 설정이 가지는 취약점을 제거하여 해커의 '공격 표면(Attack Surface)'을 줄이는 과정입니다. 단일 보안 솔루션에 의존하지 않고, 네트워크부터 애플리케이션, 데이터 계층까지 다중 방어벽을 구축하는 '심층 방어' 전략의 기반이 됩니다.

2. 엔터프라이즈 OS 하드닝 전략

기업 환경에서 가장 많이 사용되는 Linux와 Windows Server의 핵심 방어 메커니즘입니다.

Linux Hardening (MAC & PAM)

  • SELinux / AppArmor: 기존의 임의 접근 통제(DAC)를 넘어, 프로세스가 특정 파일이나 자원에 접근하는 것을 OS 커널 수준에서 강력히 통제하는 강제 접근 통제(MAC)를 활성화합니다.
  • PAM (Pluggable Authentication Modules): /etc/security/pwquality.conf 등을 수정하여 패스워드 복잡도를 강제하고, pam_tally2.so를 이용해 무차별 대입 공격(Brute Force) 시 계정을 잠급니다.
  • 불필요한 SUID/SGID 제거: 권한 상승 공격(Privilege Escalation)을 막기 위해 find / -perm -4000 명령어로 불필요한 SetUID 비트를 제거합니다.

Windows Server (AD & GPO)

  • GPO (Group Policy Object): Active Directory 환경에서 수많은 서버와 클라이언트의 보안 정책(패스워드 길이, 화면 보호기, 원격 접속 허용 IP 등)을 일괄적으로 배포하고 강제합니다.
  • SMB 보호: 랜섬웨어(WannaCry 등)의 주요 전파 경로인 SMBv1 프로토콜을 레지스트리에서 비활성화하고, SMB Signing을 강제하여 중간자 공격(MITM)을 방어합니다.
  • Credential Guard: 가상화 기반 보안(VBS)을 사용하여 메모리에 평문이나 해시 형태로 남아있는 자격 증명(NTLM, Kerberos 티켓)이 Mimikatz 등에 의해 탈취되는 것을 막습니다.

3. 모던 인프라: 컨테이너 하드닝 (Docker & K8s)

최근 IT 인프라가 클라우드 네이티브로 전환되면서, 가상화 환경(컨테이너)에 대한 하드닝이 필수 요소로 자리 잡았습니다.

docker-compose.yml (Hardening Example) 
services:
  web_app:
    image: nginx:latest
    # 1. Rootless Mode: 컨테이너 내부의 프로세스가 호스트의 root 권한을 가지지 않도록 제한
    user: "1000:1000"
    
    # 2. Read-Only Root Filesystem: 컨테이너 실행 중 악성코드 다운로드 및 변조 방지
    read_only: true
    
    # 3. Capabilities 제한: 컨테이너에 부여된 커널 권한을 모두 뺏고 필요한 것만 부여 (최소 권한의 원칙)
    cap_drop:
      - ALL
    cap_add:
      - NET_BIND_SERVICE # 80포트 바인딩만 허용
      
    # 4. 자원 고갈(DoS) 공격 방지: 메모리 및 CPU 제한
    deploy:
      resources:
        limits:
          cpus: '0.50'
          memory: 512M

4. 애플리케이션 보안: HTTP Security Headers

웹 서버(Apache, Nginx) 설정 파일에 보안 헤더를 추가하여 클라이언트(브라우저) 측의 취약점을 완화합니다.

X-Frame-Options

DENY 또는 SAMEORIGIN으로 설정하여 공격자가 보이지 않는 iframe을 띄워 클릭을 유도하는 Clickjacking 공격을 완벽히 차단합니다.

Strict-Transport-Security (HSTS)

브라우저가 해당 사이트에 접근할 때 반드시 HTTPS 암호화 통신만 사용하도록 강제하여, 중간자 공격(MITM) 및 다운그레이드 공격을 방어합니다.

Content-Security-Policy (CSP)

웹 페이지에서 로드할 수 있는 스크립트, 이미지 등의 출처(Domain)를 화이트리스트로 명시하여 XSS(크로스 사이트 스크립팅) 공격을 효과적으로 방어합니다.

5. 컴플라이언스(Compliance) 및 감사 도구

CIS Benchmarks

Center for Internet Security에서 제공하는 가장 공신력 있는 시스템 보안 설정 표준입니다. 클라우드(AWS, GCP), OS, DB 등 전방위적인 하드닝 지침을 제공합니다.

OpenSCAP

시스템 설정이 CIS Benchmarks나 NIST 표준을 준수하고 있는지 자동으로 검사(Scanning)하고 리포트를 생성해 주는 오픈소스 인프라 규정 준수 도구입니다.

Lynis

Linux/Unix 기반 시스템의 내부 깊숙한 곳까지 보안 설정을 스캔하여 취약한 항목을 찾고, 하드닝 개선 명령어를 직관적으로 제안하는 감사(Audit) 도구입니다.
