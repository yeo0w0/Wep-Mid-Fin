Offensive Domain

Pwnable (System Hacking)

바이너리의 취약점을 분석하고 메모리를 조작하여 시스템 권한을 탈취하는 익스플로잇 기술

Written by 정일재

1. 포너블과 핵심 레지스터

포너블(Pwnable)은 C/C++로 작성된 프로그램의 메모리 관리 취약점을 파고드는 기법입니다. 이를 이해하기 위해서는 x86/x64 아키텍처의 핵심 레지스터(Register)에 대한 이해가 필수적입니다.

  • RIP / EIP (Instruction Pointer): 다음에 실행할 기계어 명령의 메모리 주소를 가리킵니다. 해커의 최종 목표는 이 값을 조작하여 원하는 코드(Shellcode)를 실행하는 것입니다.
  • RBP / EBP (Base Pointer): 현재 스택 프레임의 기준점(바닥) 주소를 담고 있습니다.
  • RSP / ESP (Stack Pointer): 현재 스택의 최상단(Top) 주소를 가리키며, 데이터가 푸시/팝 될 때마다 변합니다.

2. 프로세스 메모리 구조 (Memory Layout)

프로그램 실행 시 운영체제(OS)가 RAM에 할당하는 가상 메모리 공간입니다.

Code (Text)

컴파일된 기계어 코드가 저장되며, 덮어쓰기를 막기 위해 Read/Execute 권한만 부여됩니다.

Data / BSS

전역 변수와 정적 변수(Data) 및 초기화되지 않은 전역 변수(BSS)가 위치합니다.

Heap

malloc()을 통해 동적으로 할당되는 영역으로, 낮은 주소에서 높은 주소로 확장됩니다. (Use-After-Free 취약점의 타겟)

Stack

지역 변수, 매개 변수, 반환 주소(Return Address)가 저장되며 높은 주소에서 낮은 주소로 확장됩니다. (BOF 취약점의 타겟)

3. 취약점 발생 원리 (C Code Example)

가장 대표적인 취약점인 버퍼 오버플로우(Buffer Overflow)는 입력 데이터의 길이를 검증하지 않는 함수(gets, strcpy 등)를 사용할 때 발생합니다.

vulnerable.c 
#include <stdio.h>

void vulnerable_function() {
    char buffer[64];
    
    // [!] 경고: gets() 함수는 입력 길이의 제한을 두지 않아 매우 위험합니다.
    printf("데이터를 입력하세요: ");
    gets(buffer); 
}

int main() {
    vulnerable_function();
    return 0;
}

위 코드에서 사용자가 64바이트를 초과하는 페이로드(Payload)를 입력하면, 할당된 buffer 영역을 넘어 인접한 스택 메모리를 덮어쓰게 됩니다. 이때 스택에 저장된 RBP(Base Pointer)와 RET(Return Address)를 덮어써 실행 흐름을 해커가 준비한 쉘코드(Shellcode)로 돌릴 수 있습니다.

Exploit Techniques

  • FSB (Format String Bug): printf(buffer); 처럼 포맷 스트링 인자를 사용자에게 노출할 때 발생하는 메모리 누출 및 변조 취약점.
  • ROP (Return Oriented Programming): 메모리 실행 권한 제한(NX)을 우회하기 위해, 바이너리 내부에 존재하는 pop; ret; 형태의 가젯(Gadget)들을 체인처럼 엮어 원하는 함수(예: system("/bin/sh"))를 실행하는 기법.
  • Heap Exploitation: UAF(Use-After-Free), Double Free Bug 등을 이용해 동적 메모리 관리자(ptmalloc 등)의 메타데이터를 조작하는 고급 기법.

OS Mitigations

  • ASLR (Address Space Layout Randomization): 실행 시마다 스택, 힙, 라이브러리(libc)의 주소를 무작위로 변경하여 주소 예측을 방해합니다.
  • NX (No-eXecute) / DEP: 스택과 힙 등 데이터가 저장되는 메모리 구역에 실행 권한을 박탈하여 쉘코드가 동작하지 않게 합니다.
  • Stack Canary (SSP): 버퍼와 반환 주소 사이에 무작위 값(카나리아)을 삽입하고, 함수 종료 시 변조 여부를 검사하여 BOF를 차단합니다.
  • PIE (Position Independent Executable): 라이브러리뿐만 아니라 바이너리 코드(Text 영역) 자체의 주소도 랜덤화합니다.

5. 포너블 필수 분석 도구

Pwntools

익스플로잇 스크립트 작성을 위한 Python 라이브러리. 페이로드 패킹, 리모트 소켓 통신, ROP 체인 구성을 직관적으로 돕습니다.

공식 문서 및 가이드

GDB & pwndbg

리눅스 환경의 기본 디버거(GDB)에 포너블 특화 플러그인(pwndbg)을 결합하여 레지스터와 메모리 상태를 직관적으로 추적합니다.

GitHub 레포지토리

ROPgadget

바이너리 내부에서 Return Oriented Programming 공격에 필요한 유용한 코드 조각(Gadget)들을 자동으로 검색해 주는 도구입니다.

GitHub 레포지토리
