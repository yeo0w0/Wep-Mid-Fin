Network Security

패킷 단위의 정밀한 통제와 제로 트러스트(Zero Trust) 아키텍처를 통한 인프라 방어 기술

Written by 조하율

1. 패러다임의 변화: 제로 트러스트 (Zero Trust)

전통적인 네트워크 보안은 외부망은 위험하고 내부망은 안전하다는 '경계형(Perimeter) 방어' 모델을 따랐습니다. 그러나 내부자 위협과 클라우드 도입으로 경계가 모호해지면서, "아무것도 신뢰하지 않고 항상 검증한다(Never Trust, Always Verify)"는 제로 트러스트 아키텍처(ZTA)가 글로벌 표준으로 자리 잡았습니다.

  • 마이크로 세그멘테이션 (Micro-segmentation): 네트워크를 아주 작은 단위(VM, 컨테이너 단위)로 쪼개어, 공격자가 내부에 침투하더라도 다른 구역으로 이동(Lateral Movement)하지 못하게 격리하는 핵심 기술입니다.
  • 소프트웨어 정의 경계 (SDP): 전통적인 VPN을 대체하며, 사용자의 신원(Identity)과 기기의 보안 상태를 검증한 후에만 애플리케이션 접근을 허용합니다.

2. 방화벽(Firewall)의 진화 과정

OSI 7계층에서 방화벽이 패킷을 검사하는 방식은 3세대에 걸쳐 발전해 왔습니다.

1세대: Packet Filtering

L3/L4 계층: 패킷의 출발지/목적지 IP와 Port만 단순 비교하여 차단합니다. 빠르지만, 패킷의 문맥(Context)을 이해하지 못해 IP 스푸핑에 취약합니다.

2세대: Stateful Inspection

상태 추적: TCP 3-Way Handshake와 같은 세션의 상태(State Table)를 기억합니다. 외부에서 들어오는 응답 패킷이 내부에서 요청한 것인지 확인하여 동적으로 통제합니다.

3세대: NGFW (차세대 방화벽)

L7 계층 딥 인스펙션: 애플리케이션(카카오톡, 유튜브 등) 수준의 가시성을 확보하고, IPS(침입 방지) 기능과 샌드박스, 백신 기능이 하나의 장비로 통합된 형태입니다.

3. 주요 네트워크 레벨 공격 (L2~L4)

L2: ARP Spoofing (Sniffing)

동일한 로컬 네트워크(LAN) 내에서 공격자가 자신의 MAC 주소를 라우터(Gateway)의 MAC 주소로 속여, 희생자의 모든 트래픽을 가로채고 도청하는 중간자 공격(MITM)입니다.
대응: 스위치의 ARP Inspection 활성화 및 정적(Static) ARP 테이블 구성.

L4: SYN Flooding (DDoS)

TCP의 연결 구조를 악용한 공격으로, 존재하지 않는 IP로 수많은 SYN 패킷만 보내고 ACK를 응답하지 않아 서버의 'Backlog Queue(대기 공간)'를 가득 채워 마비시키는 서비스 거부 공격입니다.
대응: TCP SYN Cookie 활성화 및 방화벽의 세션 제한 설정.
